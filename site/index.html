Local-first AI companion

Persistent, mood-aware conversation from your terminal.

SOUL pairs a fixed companion identity with SQLite-backed memory, gradual personality drift, proactive maintenance jobs, and optional Telegram or voice surfaces.

Why SOUL

Built for continuity, not disposable chat.

Persistent memory

Sessions, facts, milestones, reflections, and traces are stored locally in SQLite.

Mood-aware replies

Mood classification and companion state shape each turn before the LLM generates a response.

Slow personality drift

Resonance signals can gradually nudge humor, warmth, curiosity, and directness over time.

Multiple surfaces

The same core pipeline powers the terminal REPL, Telegram bot, and optional voice workflow.

Core surfaces

One companion, several entry points.

Terminal REPL

The main user journey: streamed turns, slash commands, traces, and local memory updates.

Telegram bot

Reuse the same orchestration pipeline for a single allowed Telegram chat.

Voice mode

Record, transcribe, and optionally speak replies with the configured voice stack.

Maintenance jobs

Run consolidation, decay, drift, reflection, and proactive follow-up jobs on local state.
